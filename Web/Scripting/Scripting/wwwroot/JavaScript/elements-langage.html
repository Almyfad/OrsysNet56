Éléments de langage

Environnement

  • Compilation JIT selon le browser
  • Garbage collector
  • Gestion de type dynamique (le type d'une variable peut changer)

Syntaxe

La syntaxe du JavaScript est basée sur celle de Java (elle même basée sur celle du langage C)

  • Instructions terminées par une virgule
  • Commentaires avec // ou /* ... */
  • Variables, constantes et littéraux
  • En l'absence de la directive "use strict", une variable se déclare de manière optionnelle avec le mot réservé var.
  • Une variable se déclarer avec var ou let.
  • Avec var, une variable déclarée dans un bloc est visible à l'extérieur.
  • Blocs délimités par des accolades.
  • Structures de contrôle if, for, while, switch, catch, etc.
  • Approche objet par le biais de fonctions avec des prototypes (classe à partir de ES6, mais ).
  • Ensemble de données avec des tableaux (Array).

Conventions

  • Blocs délimités par des accolades commençant à droite et non pas à gauche comme en C#.
  • Casse Camel pour tous les symboles, excepté pour les constructeurs avec la casse Pascal.
  • Un seul espace devant une parenthèse ouvrante, excepté pour les fonctions.

Recommandations

  • Systématiser la directive "use strict";
  • Utiliser les opérateurs === et !== (et non pas == et !=)
  • Eviter la déclaration de variables à la racine du document dites "Globales".
  • Préférer let et const à var.
  • Délimiter les chaînes de caractères externes avec "" et utiliser les apostrophes en cas d'imbrication.
  • Regrouper la déclaration des variables en tête de fonction.
  • Déclarer les fonctions avant de les invoquer.
  • Éviter les opérateurs ++ et -- (x += 1 est plus explicite)
